Markdown笔记

 2023-01-31  2290 字  markdown note learning 

Markdown笔记，用自己的方式记录Markdown语法，也可能作为cheat sheat。使用的编辑工具为Typora，如果使用其他编辑工具可能与这篇blog描述有所出入。

先来一段舒适的音乐

段落

使用空白行将文本进行分割。目前Typora等markdown编辑程序都会自动实现这个过程，不过多赘述。需要注意在Markdown中最好不要使用空格与Tab进行缩进。

换行

在一行的末尾打两个空格并回车即可创建换行。也可以使用html的换行标签<br>进行换行。

在Typora等Markdown等文本编辑器中，回车代表创建段落，shift+回车代表换行，这与平时的使用习惯可能有所不同。有时因为段落问题会导致Markdown格式很奇怪，就有可能是这个原因造成的。

分隔线

单独一行上使用三个以上的星号(*)、下划线(_)或破折号(-)创建分隔线，效果上没有区别。为保证兼容性最好前后都添加空行。

字体

基本使用：_文本_或*文本*使 文本 变为斜体，__文本__或**文本**使得 文本 变为粗体，~~文本~~为文本加删除线。

简要补充：

  1. 斜体和加粗可以同时使用，比如***文本***或**_文本_**都可以使得 文本 变为斜体加粗。更推荐**_文本_**使语法更清晰。

  2. 这些语法当然是可以跨越多个词语的，for example。

  3. 如果文本周围没有空格分隔，比如这是__一句__文本，这种情况下__下划线__会失效，请全部使用星号。

引用

在需要引用的内容前面加一个>即可，比如这句话就就在一个引用块中。

简要补充：

  1. 引用块是可以嵌套的，多加一个>即可，效果如下

    禁止套娃

    禁止套娃

  2. 引用块中支持其他的Markdown格式，这篇blog中随处可见。

代码

使用反引号(`)标识单行代码，使用一对三个反引号标识一块代码。第一组反引号后加语言名可以实现对应的语法高亮。具体实例下面会有很多的（

标题

标题分为一到六级，一级标题格式# 标题，六级标题格式###### 标题，以此类推。

简要补充：

  1. 也可以在文本下方添加任意数量的=号标识一级标题，-号标识二级标题

  2. 为了兼容考虑需要在#与标题之间使用空格隔开。

  3. 标题文本不能再加粗，但可以加斜体或删除线。

  4. 标题可以创建编号，格式为### 一个标题 {#一个编号}，可以配合下面的链接实现跳转。

链接

最基本的使用：[链接显示的文本](地址)，比如哔哩哔哩。

根据语法的不同可以进行如下区分：

超链接

[链接显示的文本](地址)，就是上面那个最基本的使用方法。

简要补充：

  1. []中的文本支持加粗斜体删除线等操作，代码格式也可以。

    fmt.Println("Hello World") ←点一下

  2. 可以为链接加title，格式为[链接显示的文本](地址 "title")，title可以用双引号、单引号或括号包裹。效果为鼠标移上去之后显示的文本。比如哔哩哔哩

  3. 可以配合标题链接使用，地址改成对应的标签就行了，比如点我回到链接标题。

网址和Email

<网址或者email>，将其变成可以点击的链接。
比如：https://www.hust.edu.cn u114514@hust.edu.cn

引用链接

第一部分：[链接显示的文本][标签]
第二部分：[标签]: 地址，这部分不会被Markdown渲染出来，大部分人喜欢将这个部分放在最末尾。 每个部分的名字都是乱起的，怎么好记怎么来。

示例：

[哔哩哔哩][1]
[1]: https://www.acfun.cn

渲染效果：

哔哩哔哩

不太简要的补充说明：

  1. 第二部分其实不需要放在第一部分之前，只不过实际使用中更习惯放在后面。
  2. 标签不区分大小写，可以包含数字、字母、标点、空格。
  3. 第一部分中链接显示的文本和标签可以使用一个空格隔开。
  4. 第二部分中的地址也是可以加title的，格式和超链接相同，比如[1]: www.acfun.cn "ニコニコ動画"。
  5. 第二部分冒号后至少需要一个空格分隔。

图片

![图片文本](图片地址)，很简单吧。[![图片文本](图片地址)](地址)就可以给图片加链接了，也很简单吧。图片文本和html的img标签里的alt功能差不多，比如图片炸了就会显示出来。

列表

无序列表

在每个列表项前加*

  • 第一项
  • 第二项
  • 第三项

有序列表

在每个列表项前加数字和句点

  1. 第一项
  2. 第二项
  3. 第三项

不太简要的补充：

  1. 有序列表的序号不一定要严格按递增顺序排序，但一定要以1.开头，最后渲染的时候仍然会渲染为1. 2. …的格式。

    1. 第一项
32. 第二项
15. 第三项
15. 第四项

    这种写法也是可以的，属于合法但有病。

  2. 列表可以嵌套。比如就可以在这个有序列表里再嵌套一个无序列表：

    • 只需要多缩进四个空格或一个tab就行了，但是经过测试三个空格也可以
    • 嵌套一般不超过三个，否则格式会很混乱

  3. 列表里也可以插入其他元素，需要将改元素多缩进四个空格或一个tab，但是经过测试三个空格也可以。比如插入一个引用：

    一个引用

  4. 或者一个图片

  5. 或者一个代码块

    import "fmt"

func main() {
	fmt.Println("Hello world!")
}

表格

格式用语言描述太难了，直接看下面的代码块：

| Syntax      | Description | Test Text     |
| :---        |    :----:   |          ---: |
| Header      | Title       | Here's this   |
| Paragraph   | Text        | And more      |

其中:---代表左对齐， :---:代表中对齐, ---:代表右对齐

一般没人会选择手打表格，请使用Markdown Tables generator或者Markdown编辑工具自带的表格生成工具来创建表格。
